常见问题急救室

如果遇到报错，请点击下方的错误信息查看解决方案。

error: remote origin already exists

原因：

本地文件夹已经关联过其他仓库，名称 'origin' 被占用。

解决方法：

git remote remove origin

然后重新添加新的 remote 地址。

弹出 "Connect to GitHub" 窗口

这是正常的认证流程。

1. 点击 Sign in with your browser。

2. 在浏览器中点击绿色的 "Authorize" 按钮。

3. 认证成功后，终端进度条会自动继续。

Password for 'https://...'

警告：

这里不能输入你的 GitHub 登录密码。

你需要输入 Personal Access Token (PAT)。

如果不习惯生成 Token，建议使用 GitHub Desktop 或 VS Code 方式，它们会自动处理认证。
